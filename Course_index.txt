1. Table Management (DDL)
    CREATE TABLE – Create a new table  
    DROP TABLE – Delete a table  
    ALTER TABLE – Modify table structure  
    TRUNCATE TABLE – Remove all records (fast)  
    RENAME TABLE – Rename a table  

2. Data Manipulation (DML)
    INSERT – Add new records    
    UPDATE – Modify existing records  
    DELETE – Remove records  
    MERGE – Insert or update based on condition (UPSERT)  

3. Data Querying (DQL)
    SELECT – Retrieve data  
    DISTINCT – Remove duplicate rows  
    WHERE – Filter rows  
    ORDER BY – Sort results  
    GROUP BY – Group rows  
    HAVING – Filter grouped data  
    LIMIT / TOP / FETCH – Restrict number of rows  
    OFFSET – Skip rows  

4. Data Control Language (DCL)
    GRANT – Grant permissions  
    REVOKE – Revoke permissions  

5. Transactions (TCL)
    BEGIN / START TRANSACTION – Start transaction  
    COMMIT – Save changes  
    ROLLBACK – Undo changes  
    SAVEPOINT – Create rollback point  

6. Indexing
    CREATE INDEX – Create an index  
    DROP INDEX – Remove an index  
    ALTER INDEX – Modify an index  

7. joins -very important
    left join
    right join
    inner join
    outer join
    cross join


8. Partitioning -
    CREATE PARTITION – Create a partition  
    DROP PARTITION – Remove a partition  
    ALTER PARTITION – Modify a partition  

9. Clustering  -- less important
    CREATE CLUSTER – Create a cluster  
    DROP CLUSTER – Remove a cluster  
    ALTER CLUSTER – Modify a cluster  

10. Materialized Views
    CREATE MATERIALIZED VIEW – Create a materialized view  
    DROP MATERIALIZED VIEW – Remove a materialized view  
    ALTER MATERIALIZED VIEW – Modify a materialized view  

11. Triggers -- less important
    CREATE TRIGGER – Create a trigger  
    DROP TRIGGER – Remove a trigger  
    ALTER TRIGGER – Modify a trigger  

12. Functions
    CREATE FUNCTION – Create a function  
    DROP FUNCTION – Remove a function  
    ALTER FUNCTION – Modify a function  

13. Synonyms --less important
    CREATE SYNONYM – Create a synonym  
    DROP SYNONYM – Remove a synonym  
    ALTER SYNONYM – Modify a synonym  


14. Views
    CREATE VIEW – Create a view  
    DROP VIEW – Remove a view  
    ALTER VIEW – Modify a view  

15. Procedures -- less important
    CREATE PROCEDURE – Create a procedure  
    DROP PROCEDURE – Remove a procedure  
    ALTER PROCEDURE – Modify a procedure  


16. Packages --less important
    CREATE PACKAGE – Create a package

-----------DataWare House Concepts-----------

1. Basics
    Data Warehouse – Centralized repository for integrated data  
    OLTP – Online Transaction Processing (day-to-day operations)  
    OLAP – Online Analytical Processing (reporting & analysis)  
    Subject-Oriented – Organized around key business subjects  
    Integrated – Data from multiple sources  
    Time-Variant – Historical data stored over time  
    Non-Volatile – Read-only, no frequent updates  

2. Data Warehouse Architecture
    Source Systems – OLTP, files, APIs, external feeds  
    Staging Area – Temporary storage for raw data  
    ETL – Extract, Transform, Load  
    ELT – Extract, Load, Transform  
    Data Warehouse – Central storage  
    Data Mart – Department-level subset  
    Presentation Layer – Reporting & BI tools  

3. Data Modeling
    Fact Table – Stores measurable business data  
    Dimension Table – Descriptive attributes  
    Surrogate Key – System-generated key  
    Natural Key – Business key  
    Grain – Level of detail in fact table  
    Degenerate Dimension – Dimension without table  

4. Schema Types
    Star Schema – One fact, many dimensions  
    Snowflake Schema – Normalized dimensions  
    Galaxy Schema – Multiple fact tables  
    Flat Schema – Denormalized structure  

5. Fact Tables
    Transaction Fact – One row per transaction  
    Snapshot Fact – Periodic data capture  
    Accumulating Snapshot – Lifecycle tracking  
    Factless Fact – Event tracking without measures   

6. Slowly Changing Dimensions (SCD)
    Type 0 – No change allowed  
    Type 1 – Overwrite old data  
    Type 2 – New row with history  
    Type 3 – Limited history (new column)  
    Type 4 – Separate history table  
  
7. ETL / ELT Concepts
    Extraction – Pull data from sources  
    Transformation – Cleanse & enrich data  
    Loading – Insert into warehouse  
    Full Load – Reload entire data  
    Incremental Load – Load changes only  
    CDC – Change Data Capture  
    Watermark – Track last processed record 

8. Data Quality
    Accuracy – Correct data  
    Completeness – No missing values  
    Consistency – Same data everywhere  
    Timeliness – Data availability on time  
    Validity – Conforms to rules  
    Uniqueness – No duplicates  

9. Performance Optimization
    Indexing – Speed up queries  
    Partitioning – Split large tables  
    Clustering – Data organization  
    Materialized View – Precomputed results  
    Aggregates – Pre-summarized data  

10. Data Governance & Security
    Data Lineage – Track data flow  
    Data Ownership – Responsibility definition  
    Access Control – Role-based security  
    Masking – Hide sensitive data  
    Auditing – Track data access
